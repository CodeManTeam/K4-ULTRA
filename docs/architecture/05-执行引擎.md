## 2. Registry 注册中心

### 2.1 功能说明

Registry 是执行引擎的核心，管理所有 Domain Function 的注册和查找。

### 2.2 数据结构

```javascript
class RegistryImpl {
    // Domain Function 索引映射
    // key: 积木类型名
    // value: 在 domain_function_list 中的索引
    domain_function_index = {
        "motion_movesteps": 0,
        "control_repeat": 1,
        "event_whenflagclicked": 2,
        // ...
    };
    
    // Domain Function 列表
    // 按注册顺序存储所有 Domain Function
    domain_function_list = [
        function motion_movesteps(args) { ... },
        function control_repeat(args) { ... },
        function event_whenflagclicked(args) { ... },
        // ...
    ];
    
    // Domain Function 类型名
    // 与 domain_function_list 一一对应
    domain_function_types = [
        "motion_movesteps",
        "control_repeat",
        "event_whenflagclicked",
        // ...
    ];
}
```

### 2.3 注册方法

```javascript
/**
 * 注册 Domain Function
 * @param {Object} config
 * @param {string} config.namespace - 命名空间（通常为空字符串）
 * @param {string} config.id - Domain Function ID（积木类型名）
 * @param {Function} config.domain_function - Domain Function 实现
 */
register({ namespace, id, domain_function }) {
    // 生成唯一键
    var key = namespace ? namespace + "." + id : id;
    
    // 检查是否已注册（去重）
    if (this.domain_function_index[key] != null) {
        return;  // 已注册，跳过
    }
    
    // 分配索引
    var index = this.domain_function_list.length;
    
    // 注册
    this.domain_function_index[key] = index;
    this.domain_function_list.push(domain_function);
    this.domain_function_types.push(key);
}
```

### 2.4 查找方法

```javascript
/**
 * 获取 Domain Function 索引
 * @param {string} type - 积木类型名
 * @returns {number} 索引值，未找到返回 undefined
 */
get_domain_function_index(type) {
    return this.domain_function_index[type];
}

/**
 * 获取所有 Domain Function 列表
 * @returns {Array} Domain Function 数组
 */
get_domain_function_list() {
    return this.domain_function_list;
}

/**
 * 获取所有 Domain Function 类型名
 * @returns {Array} 类型名数组
 */
get_domain_function_types() {
    return this.domain_function_types;
}
```

### 2.5 注册流程

```
Domain Function 注册
  │
  ├─→ 方式一：在 get_domain_functions 的 v 对象中注入
  │     └─→ Heart IIFE 自动注册
  │
  ├─→ 方式二：通过 Registry.register() 手动注册
  │
  └─→ 方式三：通过 Bridge API 注册（扩展系统）
```

## 3. OptiCompiler 字节码编译器

### 3.1 功能说明

OptiCompiler 将积木树编译为字节码程序（OptiProgram），供 OptiRunner 执行。

### 3.2 编译流程

```
积木树 (Blockly Blocks)
  │
  ├─→ 1. 获取起始积木（帽子积木）
  │
  ├─→ 2. 遍历积木链
  │     │
  │     ├─→ 对每个积木:
  │     │     ├─→ 检查积木类型
  │     │     ├─→ 如果是 Domain Block:
  │     │     │     ├─→ 查找 Domain Function 索引
  │     │     │     ├─→ 生成 call_domain_function 指令
  │     │     │     └─→ 处理参数
  │     │     ├─→ 如果是控制流积木:
  │     │     │     ├─→ 生成 jump 指令
  │     │     │     └─→ 处理条件分支
  │     │     └─→ 如果是其他积木:
  │     │           └─→ 生成相应指令
  │     │
  │     └─→ 处理下一个积木
  │
  └─→ 3. 输出字节码程序 (OptiProgram)
```

### 3.3 字节码指令集

| 指令 | 格式 | 说明 |
|------|------|------|
| `X.call_domain_function` | `X.call_domain_function(type, index, id, dynamic, entity)` | 调用 Domain Function |
| `X.AB` | `X.AB(type, index, id, dynamic, entity)` | 调用 Domain Function（简写） |
| `X.jump` | `X.jump(target)` | 无条件跳转 |
| `X.jump_if` | `X.jump_if(condition, target)` | 条件跳转 |
| `X.push` | `X.push(value)` | 值入栈 |
| `X.pop` | `X.pop()` | 值出栈 |
| `X.yield` | `X.yield()` | 协程让出 |
| `X.stop` | `X.stop()` | 停止执行 |
| `X.return` | `X.return(value)` | 返回值 |

### 3.4 编译示例

**积木**：`移动 10 步`

**积木定义**：
```javascript
{
    type: "motion_movesteps",
    id: "block_123",
    inputs: {
        STEPS: { block: { type: "math_number", fields: { NUM: 10 } } }
    }
}
```

**编译结果**：
```javascript
[
    "X.call_domain_function",
    "('motion_movesteps',",
    "0,",                    // domain function 索引
    "'block_123',",         // 积木 ID
    "null,",                // 动态数据
    "entity_456);"          // 目标实体
]
```

### 3.5 Domain Function 编译

```javascript
/**
 * 编译 Domain Function 调用
 * @param {Object} block - 积木对象
 * @returns {Array} 字节码指令
 */
compile_domain_functions(block) {
    // 获取 Domain Function 索引
    var index = this.registry.get_domain_function_index(block.type);
    
    // 检查是否存在
    if (index == null) {
        if (!this.config.get().ignore_missing_domain_function) {
            throw this.ohno.client.missing_domain_function({
                block_type: block.type
            });
        }
        return null;
    }
    
    // 生成字节码
    return [
        this.keyword.call_domain_function,
        "('", block.type, "', ",
        index.toString(), ", '",
        block.id, "', ",
        this.opt_params(block.dynamic_data_id), ", ",
        this.state.current_target_entity(), ");"
    ];
}
```

## 4. OptiRunner 字节码解释器

### 4.1 功能说明

OptiRunner 执行 OptiCompiler 编译产生的字节码程序。

### 4.2 执行流程

```
字节码程序 (OptiProgram)
  │
  ├─→ 1. 创建执行帧 (OptiFrame)
  │
  ├─→ 2. 初始化执行上下文
  │     ├─→ 设置程序计数器 (PC)
  │     ├─→ 初始化操作数栈
  │     └─→ 初始化局部变量
  │
  ├─→ 3. 逐条执行指令:
  │     │
  │     ├─→ call_domain_function:
  │     │     ├─→ 用索引从 domain_function_list 获取函数
  │     │     ├─→ 准备参数
  │     │     ├─→ 调用函数
  │     │     ├─→ 处理返回值
  │     │     └─→ 推入栈
  │     │
  │     ├─→ jump:
  │     │     └─→ 修改 PC
  │     │
  │     ├─→ jump_if:
  │     │     ├─→ 弹出条件值
  │     │     └─→ 如果为真，修改 PC
  │     │
  │     ├─→ yield:
  │     │     ├─→ 保存执行状态
  │     │     └─→ 返回，等待下一帧继续
  │     │
  │     └─→ stop:
  │           └─→ 停止执行
  │
  └─→ 4. 处理执行结果
```

### 4.3 执行帧 (OptiFrame)

```javascript
class OptiFrame {
    // 程序
    program = null           // 字节码程序
    
    // 执行状态
    pc = 0                   // 程序计数器
    stack = []               // 操作数栈
    variables = {}           // 局部变量
    
    // 源映射（用于调试）
    source_map_rbid = ""     // 源积木 ID
    source_map_entity = ""   // 源实体 ID
    
    // 协程状态
    is_yielding = false      // 是否正在 yield
    yield_value = null       // yield 值
}
```

### 4.4 Domain Function 调用

```javascript
/**
 * 调用 Domain Function
 * @param {string} block_id - 积木 ID
 * @param {number} func_index - Domain Function 索引
 * @param {string} block_type - 积木类型
 * @param {*} dynamic_data - 动态数据
 * @param {string} entity_id - 目标实体 ID
 * @returns {*} Domain Function 返回值
 */
step_call_domain_function(block_id, func_index, block_type, dynamic_data, entity_id) {
    try {
        // 获取 Domain Function
        var func = this.domain_functions[func_index];
        
        // 准备参数
        var args = this.prepare_args(block_id, dynamic_data);
        
        // 调用函数
        var result = func(
            args,
            this.original_identities.interpreter_id,
            entity_id,
            this.domain_function_util
        );
        
        // 处理返回值
        return result;
    } catch (error) {
        // 错误处理
        throw this.extend_domain_function_call_error(
            error, block_id, block_type,
            this.current_frame.source_map_rbid,
            this.current_frame.source_map_entity
        );
    }
}
```

### 4.5 协程支持

OptiRunner 支持协程（yield），用于：

- **等待操作**：如 `wait 1 秒`
- **动画播放**：如 `移动 10 步`（动画）
- **事件等待**：如 `当点击时`

```javascript
// yield 处理
if (result && result.type === 'yield') {
    // 保存执行状态
    this.current_frame.is_yielding = true;
    this.current_frame.yield_value = result.value;
    
    // 返回，等待下一帧继续
    return { type: 'yield', interpreter_id: this.id };
}
```

## 5. Domain Function 运行时函数

### 5.1 功能说明

Domain Function 是积木的实际执行逻辑，每个积木类型对应一个 Domain Function。

### 5.2 函数签名

```javascript
/**
 * Domain Function 签名
 * @param {Object} args - 积木参数对象
 * @param {string} interpreter_id - 解释器 ID
 * @param {string} entity_id - 目标实体 ID
 * @param {Object} util - 工具函数对象
 * @returns {*} 返回值（值积木必须返回）
 */
function domain_function_name(args, interpreter_id, entity_id, util) {
    // args 结构:
    //   - args.PARAM_NAME = 输入参数值
    //   - args.DROPDOWN = 下拉框值
    //   - args.FIELD = 字段值
    
    // util 方法:
    //   - util.create_domain_function_error(error) = 创建错误
    
    try {
        // 业务逻辑
        var result = doSomething(args.PARAM);
        
        // 值积木必须返回结果
        return result;
    } catch(err) {
        // 错误处理，不要抛异常！
        console.error('[DomainFunction] Error:', err);
        return default_value;
    }
}
```

### 5.3 参数访问

```javascript
// 积木定义
{
    type: "my_block",
    message0: "读取文件 %1 模式 %2",
    args0: [
        { type: "input_value", name: "PATH", check: "String" },
        { type: "field_dropdown", name: "MODE", options: [["读取","read"],["写入","write"]] }
    ]
}

// Domain Function
function my_block(args, interpreter_id, entity_id, util) {
    var path = args.PATH;      // 获取输入参数（值积木的返回值）
    var mode = args.MODE;      // 获取下拉框值 ("read" 或 "write")
    
    // 业务逻辑
    if (mode === "read") {
        return readFile(path);
    } else {
        writeFile(path, data);
    }
}
```

### 5.4 注册方式

**方式一：在 get_domain_functions 的 v 对象中注入**

```javascript
// 搜索 get_domain_functions（约150140行）
function get_domain_functions() {
    var p = {
        // 辅助函数（不会被注册）
        year: function() { return new Date().getFullYear(); },
        month: function() { return new Date().getMonth() + 1; }
    };
    
    var v = {
        // 核心 Domain Functions（会被注册）
        when: function(args) { ... },
        wait: function(args) { ... },
        
        // ★ 你的新函数必须注入到这里 ★
        my_function: function(args) {
            return doSomething(args.PARAM);
        }
    };
    
    return v;  // 注意：返回 v，不是 p！
}
```

```javascript

```

**方式三：通过 Bridge API 注册（扩展系统）**


| 规则 | 说明 | 违反后果 |

## 6. RuntimeManager 运行时管理器

### 6.1 功能说明

RuntimeManager 管理所有解释器实例的生命周期。

### 6.2 核心职责

- **创建解释器**：为每个帽子积木创建解释器
- **管理解释器**：启动、暂停、恢复、停止
- **调度执行**：按优先级调度解释器
- **处理事件**：广播、消息传递

### 6.3 解释器生命周期

```
帽子积木触发
  │
  ├─→ 创建 Interpreter
  │
  ├─→ 编译积木树
  │
  ├─→ 执行字节码
  │     ├─→ 正常执行
  │     ├─→ yield（等待）
  │     └─→ 完成
  │
  └─→ 销毁 Interpreter
```

## 7. DebugRunner 调试模式

### 7.1 功能说明

DebugRunner 是 OptiRunner 的调试版本，支持断点调试和单步执行。

### 7.2 调试功能

| 功能 | 说明 |
|------|------|
| **断点调试** | 在积木上设置断点 |
| **单步执行** | 逐个积木执行 |
| **变量监视** | 查看变量值 |
| **调用栈查看** | 查看函数调用栈 |
| **执行暂停** | 暂停/恢复执行 |

### 7.3 调试流程

```
用户开启调试模式
  │
  ├─→ 创建 DebugRunner
  │
  ├─→ 设置断点
  │
  ├─→ 执行积木:
  │     ├─→ 检查是否命中断点
  │     ├─→ 如果命中:
  │     │     ├─→ 暂停执行
  │     │     ├─→ 显示调试信息
  │     │     └─→ 等待用户操作
  │     ├─→ 单步执行
  │     └─→ 继续执行
  │
  └─→ 显示调试结果
```

### 7.4 调试 API

```javascript
// 创建调试 Runner
var debugRunner = debugRunnerFactory.create(
    runtime, block_pool, domain_functions, ...
);

// 设置断点
debugRunner.set_breakpoint(block_id);

// 单步执行
debugRunner.step();

// 继续执行
debugRunner.continue();

// 获取变量
debugRunner.get_variable(name);

// 获取调用栈
debugRunner.get_call_stack();
```

```javascript

    ignore_missing_domain_function: false,
```

## 附录：常见问题

### Q1: 积木可见但不执行

**原因**：Domain Function 未注册

**解决**：
1. 检查是否注入到了 v 对象
2. 检查是否通过 Registry 注册
3. 检查函数名是否与积木 type 一致

### Q2: "调用的函数不存在" 错误

**原因**：Domain Function 索引错误

**解决**：
1. 检查 Registry 注册是否成功
2. 检查积木类型名是否正确
3. 检查是否有重复注册

### Q3: 执行卡死

**原因**：无限循环或死锁

**解决**：
1. 检查循环条件
2. 检查是否有 yield
3. 检查最大调用栈配置

### Q4: 协程不工作

**原因**：yield 处理错误

**解决**：
1. 检查 Domain Function 是否正确返回 yield
2. 检查 RuntimeManager 调度
3. 检查帧率设置
