## 1. 架构概述

K4 的积木执行系统采用**双层架构**：

```
┌─────────────────────────────────────────────────────────────┐
│                    K4 积木执行双层架构                        │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  ┌─────────────┐     ┌──────────────┐     ┌──────────────┐ │
│  │ 积木定义     │ ──→ │ JS Generator  │ ──→ │ 代码预览      │ │
│  │ (JSON)      │     │ (代码生成器)   │     │ ("看代码")    │ │
│  └─────────────┘     └──────────────┘     └──────────────┘ │
│         │                                               ▲   │
│         │              ★ 主执行路径 ★                    │   │
│         ↓                                               │   │
│  ┌─────────────┐     ┌──────────────┐     ┌──────────────┐ │
│  │ 积木定义     │ ──→ │ OptiCompiler  │ ──→ │ OptiRunner   │ │
│  │ (JSON)      │     │ (字节码编译器)  │     │ (字节码解释器)│ │
│  └─────────────┘     └──────────────┘     └──────────────┘ │
│                              │                    │        │
│                              ▼                    ▼        │
│                    ┌──────────────────────────────┐        │
│                    │     Registry (单例)           │        │
│                    │  domain_function_index[]      │        │
│                    │  domain_function_list[]       │        │
│                    └──────────────────────────────┘        │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 1.1 两条执行路径

| 路径 | 用途 | 技术 |
|------|------|------|
| **JS Generator** | 代码预览（"看代码"功能） | 生成可读 JS 代码字符串 |
| **OptiCompiler + OptiRunner** | 实际执行 | 编译为字节码并执行 |

### 1.2 为什么需要双层架构？

| 需求 | JS Generator | OptiCompiler/OptiRunner |
|------|--------------|-------------------------|
| 代码可读性 | ✓ 高 | ✗ 低（字节码） |
| 执行效率 | ✗ 低 | ✓ 高 |
| 调试支持 | ✗ 有限 | ✓ 完整 |
| 错误处理 | ✗ 基础 | ✓ 完善 |
| 协程支持 | ✗ 不支持 | ✓ 支持 |

## 2. JS Generator（代码生成器）

### 2.1 功能说明

JS Generator 用于"看代码"功能，将积木转换为可读的 JavaScript 代码字符串。

**重要**：JS Generator 仅用于代码展示，不参与实际执行。

### 2.2 注册方式

```javascript
// 搜索 blink.js_generator.register
gen.register("block_type_name", function(block) {
    // block 是积木实例
    
    // 获取参数值
    var param = gen.value_to_code(block, "PARAM_NAME", gen.ORDER_NONE) || "default";
    
    // 值积木：返回 [代码字符串, 优先级]
    return ["(expression)", gen.ORDER_FUNCTION_CALL];
    
    // 语句积木：返回代码字符串（以 \n 结尾）
    return "doSomething();\n";
});
```

### 2.3 返回格式

| 积木类型 | 返回格式 | 示例 |
|---------|---------|------|
| 值积木（有 output） | `[code_string, order]` | `["(Date.now())", gen.ORDER_FUNCTION_CALL]` |
| 语句积木（无 output） | `code_string`（以 \n 结尾） | `"doSomething();\n"` |

### 2.4 常用 API

```javascript
// 获取参数代码
gen.value_to_code(block, "PARAM", gen.ORDER_NONE)

// 获取语句代码
gen.statement_to_code(block, "SUBSTACK")

// 优先级常量
gen.ORDER_FUNCTION_CALL  // 函数调用
gen.ORDER_NONE           // 无优先级
gen.ORDER_ATOMIC         // 原子表达式
```

### 2.5 示例：移动积木

```javascript
// 积木定义
{
    type: "motion_movesteps",
    message0: "移动 %1 步",
    args0: [
        { type: "input_value", name: "STEPS", check: "Number" }
    ],
    previousStatement: null,
    nextStatement: null
}

// JS Generator
gen.register("motion_movesteps", function(block) {
    var steps = gen.value_to_code(block, "STEPS", gen.ORDER_NONE) || "10";
    return "move(" + steps + ");\n";
});

// 生成的代码
"move(10);\n"
```

### 2.6 代码预览位置

代码预览功能的关键代码位置：

- Generator 注册：搜索 `blink.js_generator.register`
- 代码预览面板：搜索 `CodePreview` 组件
- 代码格式化：搜索 `js_beautify`

## 3. OptiCompiler（字节码编译器）

### 3.1 功能说明

OptiCompiler 将积木树编译为字节码程序（OptiProgram），供 OptiRunner 执行。

### 3.2 编译流程

```
积木树 (Blockly Blocks)
  │
  ├─→ 遍历积木
  │
  ├─→ 对每个积木:
  │     ├─→ 查找 domain function 索引
  │     ├─→ 生成字节码指令
  │     └─→ 处理连接关系
  │
  └─→ 输出字节码程序 (OptiProgram)
```

### 3.3 字节码指令集

| 指令 | 说明 | 示例 |
|------|------|------|
| `X.call_domain_function` | 调用 Domain Function | `X.call_domain_function('type', index, id, ...)` |
| `X.AB` | 调用 Domain Function（简写） | `X.AB('type', index, id, ...)` |
| `X.jump` | 跳转 | `X.jump(target)` |
| `X.jump_if` | 条件跳转 | `X.jump_if(condition, target)` |
| `X.push` | 入栈 | `X.push(value)` |
| `X.pop` | 出栈 | `X.pop()` |
| `X.yield` | 协程让出 | `X.yield()` |
| `X.stop` | 停止 | `X.stop()` |

### 3.4 编译示例

**积木**：`移动 10 步`

**编译结果**：
```javascript
[
    "X.call_domain_function",
    "('motion_movesteps',",
    "0,",                    // domain function 索引
    "'block_123',",         // 积木 ID
    "null,",                // 动态数据
    "entity_456);"          // 目标实体
]
```

### 3.5 关键代码位置

- 编译器实现：搜索 `OptiCompilerImpl`
- Domain Function 编译：搜索 `compile_domain_functions`
- 索引查找：搜索 `get_domain_function_index`

### 3.6 编译优化

OptiCompiler 进行以下优化：

- **常量折叠**：编译时计算常量表达式
- **死代码消除**：移除不可达代码
- **内联优化**：内联简单的 Domain Function
- **缓存优化**：缓存编译结果

## 4. OptiRunner（字节码解释器）

### 4.1 功能说明

OptiRunner 执行 OptiCompiler 编译产生的字节码程序。

### 4.2 执行流程

```
字节码程序 (OptiProgram)
  │
  ├─→ 创建执行帧 (OptiFrame)
  │
  ├─→ 逐条执行指令:
  │     ├─→ call_domain_function:
  │     │     ├─→ 用索引从 domain_function_list 获取函数
  │     │     ├─→ 调用函数，传入参数
  │     │     └─→ 处理返回值
  │     │
  │     ├─→ jump / jump_if:
  │     │     └─→ 控制流跳转
  │     │
  │     ├─→ yield:
  │     │     └─→ 协程让出，保存状态
  │     │
  │     └─→ stop:
  │           └─→ 停止执行
  │
  ├─→ 处理协程 (yield)
  │
  └─→ 处理错误
```

### 4.3 执行帧 (OptiFrame)

```javascript
class OptiFrame {
    program = null           // 字节码程序
    pc = 0                   // 程序计数器
    stack = []               // 操作数栈
    variables = {}           // 局部变量
    source_map_rbid = ""     // 源映射
    source_map_entity = ""   // 源实体
}
```

### 4.4 Domain Function 调用

```javascript
// OptiRunner 调用 Domain Function
step_call_domain_function(block_id, func_index, block_type, dynamic_data, entity_id) {
    // 获取 Domain Function
    var func = this.domain_functions[func_index];
    
    // 调用函数
    var result = func(args, interpreter_id, entity_id, util);
    
    // 处理返回值
    return result;
}
```

### 4.5 协程支持

OptiRunner 支持协程（yield），用于：

- **等待操作**：如 `wait 1 秒`
- **动画播放**：如 `移动 10 步`（动画）
- **事件等待**：如 `当点击时`

```javascript
// 协程示例
function* moveSteps(steps) {
    for (var i = 0; i < steps; i++) {
        moveActor(1);
        yield;  // 让出执行权，下一帧继续
    }
}
```

### 4.6 关键代码位置

- Runner 实现：搜索 `OptiRunner`
- Domain Function 调用：搜索 `step_call_domain_function`
- 协程处理：搜索 `yield`

## 5. Domain Function（运行时函数）

### 5.1 功能说明

Domain Function 是积木的实际执行逻辑，每个积木类型对应一个 Domain Function。

### 5.2 函数签名

```javascript
function domain_function_name(args, interpreter_id, entity_id, util) {
    // args = 积木参数对象
    //   - args.PARAM_NAME = 参数值
    //   - args.DROPDOWN = 下拉框值
    //
    // interpreter_id = 解释器 ID
    // entity_id = 目标实体 ID（角色/场景）
    // util = 工具函数对象
    //   - util.create_domain_function_error(error) = 创建错误
    
    try {
        // 业务逻辑
        var result = doSomething(args.PARAM);
        
        // 值积木必须返回结果
        return result;
    } catch(err) {
        // 错误处理，不要抛异常！
        console.error(err);
        return default_value;
    }
}
```

### 5.3 参数访问

```javascript
// 积木定义
{
    type: "my_block",
    args0: [
        { type: "input_value", name: "PATH", check: "String" },
        { type: "field_dropdown", name: "MODE", options: [["打开","open"],["保存","save"]] }
    ]
}

// Domain Function
function my_block(args) {
    var path = args.PATH;      // 获取输入参数
    var mode = args.MODE;      // 获取下拉框值 ("open" 或 "save")
    
    // 业务逻辑
}
```

### 5.4 注册方式

**方式一：在 get_domain_functions 的 v 对象中注入**

```javascript
// 搜索 get_domain_functions（约150140行）
function get_domain_functions() {
    var p = {
        // 辅助函数（不会被注册）
        year: function() { return new Date().getFullYear(); },
        month: function() { return new Date().getMonth() + 1; }
    };
    
    var v = {
        // 核心 Domain Functions（会被注册）
        when: function(args) { ... },
        wait: function(args) { ... },
        
        // ★ 你的新函数必须注入到这里 ★
        my_function: function(args) {
            return doSomething(args.PARAM);
        }
    };
    
    return v;  // 注意：返回 v，不是 p！
}
```

**方式二：通过 Registry 注册**

```javascript
// 在 Heart IIFE 中注册
var _reg = _bbp.registry;
_reg.register({
    namespace: "",
    id: "my_function",
    domain_function: function(args) {
        return doSomething(args.PARAM);
    }
});
```

### 5.5 重要规则

| 规则 | 说明 |
|------|------|
| **必须注入到 v 对象** | 注入到 p 对象不会生效 |
| **不要 return p** | 永远保持 return v |
| **使用 require()** | 不要用 window.FS / window.System |
| **用 try/catch 包裹** | 不要抛异常，会中断程序 |
| **值积木必须 return** | 有 output 的积木需返回结果 |

## 6. Registry（注册中心）

### 6.1 功能说明

Registry 是执行引擎的核心，管理所有 Domain Function 的注册和查找。

### 6.2 数据结构

```javascript
class RegistryImpl {
    // Domain Function 索引映射
    domain_function_index = {
        "motion_movesteps": 0,
        "control_repeat": 1,
        "event_whenflagclicked": 2,
        // ...
    };
    
    // Domain Function 列表
    domain_function_list = [
        function motion_movesteps(args) { ... },
        function control_repeat(args) { ... },
        function event_whenflagclicked(args) { ... },
        // ...
    ];
    
    // Domain Function 类型名
    domain_function_types = [
        "motion_movesteps",
        "control_repeat",
        "event_whenflagclicked",
        // ...
    ];
}
```

### 6.3 注册方法

```javascript
register({ namespace, id, domain_function }) {
    // 生成唯一键
    var key = namespace ? namespace + "." + id : id;
    
    // 检查是否已注册
    if (this.domain_function_index[key] != null) {
        return;  // 已注册，跳过
    }
    
    // 分配索引
    var index = this.domain_function_list.length;
    
    // 注册
    this.domain_function_index[key] = index;
    this.domain_function_list.push(domain_function);
    this.domain_function_types.push(key);
}
```

### 6.4 查找方法

```javascript
// 获取索引
get_domain_function_index(type) {
    return this.domain_function_index[type];
}

// 获取函数
get_domain_function_list() {
    return this.domain_function_list;
}

// 获取类型名
get_domain_function_types() {
    return this.domain_function_types;
}
```

### 6.5 关键代码位置

- Registry 实现：搜索 `RegistryImpl`
- 注册方法：搜索 `registry.register`
- 单例绑定：搜索 `BINDING.Registry`

## 7. 执行流程对比

### 7.1 JS Generator 流程（代码预览）

```
用户点击"看代码"
  │
  ├─→ 获取积木树
  │
  ├─→ 遍历积木
  │
  ├─→ 对每个积木:
  │     ├─→ 查找 Generator
  │     └─→ 调用 Generator，生成代码字符串
  │
  ├─→ 拼接所有代码
  │
  └─→ 显示代码预览
```

### 7.2 OptiCompiler/OptiRunner 流程（实际执行）

```
用户点击运行
  │
  ├─→ 获取积木树
  │
  ├─→ OptiCompiler 编译:
  │     ├─→ 遍历积木
  │     ├─→ 查找 Domain Function 索引
  │     └─→ 生成字节码
  │
  ├─→ OptiRunner 执行:
  │     ├─→ 创建执行帧
  │     ├─→ 逐条执行指令
  │     ├─→ 调用 Domain Function
  │     └─→ 处理协程
  │
  └─→ 更新状态，显示结果
```

### 7.3 对比总结

| 方面 | JS Generator | OptiCompiler/OptiRunner |
|------|--------------|-------------------------|
| **输入** | 积木树 | 积木树 |
| **输出** | 代码字符串 | 字节码 → 执行结果 |
| **用途** | 代码预览 | 实际执行 |
| **性能** | 低（字符串拼接） | 高（字节码执行） |
| **调试** | 有限 | 完整 |
| **协程** | 不支持 | 支持 |

## 8. 调试模式

### 8.1 DebugRunner

DebugRunner 是 OptiRunner 的调试版本，支持：

- **断点调试**：在积木上设置断点
- **单步执行**：逐个积木执行
- **变量监视**：查看变量值
- **调用栈查看**：查看函数调用栈

### 8.2 调试流程

```
用户开启调试模式
  │
  ├─→ 创建 DebugRunner
  │
  ├─→ 设置断点
  │
  ├─→ 执行积木:
  │     ├─→ 检查是否命中断点
  │     ├─→ 如果命中，暂停执行
  │     ├─→ 等待用户操作
  │     └─→ 继续执行
  │
  └─→ 显示调试信息
```

### 8.3 调试 API

```javascript
// 创建调试 Runner
var debugRunner = new DebugRunner(
    util, ohno, event_bus,
    runtime, block_pool,
    domain_functions, ...
);

// 设置断点
debugRunner.set_breakpoint(block_id);

// 单步执行
debugRunner.step();

// 继续执行
debugRunner.continue();

// 获取变量
debugRunner.get_variable(name);

// 获取调用栈
debugRunner.get_call_stack();
```

### 8.4 关键代码位置

- DebugRunner 实现：搜索 `DebugRunner`
- 调试面板：搜索 `Debugger` 组件
- 断点管理：搜索 `breakpoint`

## 附录：常见问题

### Q1: 积木可见但不执行

**原因**：Domain Function 未注册

**解决**：
1. 检查是否注入到了 v 对象
2. 检查是否通过 Registry 注册
3. 检查函数名是否与积木 type 一致

### Q2: 积木执行但无效果

**原因**：Domain Function 内部错误被 try/catch 吞掉

**解决**：
1. 在 Domain Function 入口添加日志
2. 临时去掉 try/catch 测试
3. 检查使用的 API 是否可用

### Q3: "看代码"显示不正确

**原因**：JS Generator 注册错误

**解决**：
1. 检查 Generator 是否注册
2. 检查返回格式是否正确
3. 检查参数获取是否正确
