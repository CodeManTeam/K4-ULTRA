## 1. 架构总览

K4 Ultra 采用经典的分层架构，从上到下分为五层：

```
┌─────────────────────────────────────────────────────────────────┐
│                        K4 Ultra 架构                            │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  ┌───────────────────────────────────────────────────────────┐  │
│  │                    UI 层 (React)                          │  │
│  │  工具栏 │ 舞台 │ 属性面板 │ 代码预览 │ 调试面板           │  │
│  └───────────────────────────────────────────────────────────┘  │
│                              ↕                                  │
│  ┌───────────────────────────────────────────────────────────┐  │
│  │                  状态管理层 (Redux)                       │  │
│  │  Store │ Actions │ Reducers │ Sagas                      │  │
│  └───────────────────────────────────────────────────────────┘  │
│                              ↕                                  │
│  ┌───────────────────────────────────────────────────────────┐  │
│  │                  Blockly 引擎层                           │  │
│  │  Workspace │ Toolbox │ Flyout │ Blocks │ Events          │  │
│  └───────────────────────────────────────────────────────────┘  │
│                              ↕                                  │
│  ┌───────────────────────────────────────────────────────────┐  │
│  │                  执行引擎层 (Blink)                       │  │
│  │  Registry │ OptiCompiler │ OptiRunner │ Domain Functions  │  │
│  └───────────────────────────────────────────────────────────┘  │
│                              ↕                                  │
│  ┌───────────────────────────────────────────────────────────┐  │
│  │                  基础设施层                               │  │
│  │  DI容器 │ EventBus │ 错误处理 │ 国际化 │ 主题            │  │
│  └───────────────────────────────────────────────────────────┘  │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 2. UI 层

### 2.1 职责

UI 层负责：
- 渲染用户界面
- 处理用户交互
- 展示编辑器状态
- 提供可视化反馈

### 2.2 技术栈

- **React 17.x**：UI 框架
- **React-Redux**：连接 Redux Store
- **CSS Modules**：样式隔离

### 2.3 主要组件

| 组件 | 功能 | 位置 |
|------|------|------|
| `App` | 根组件 | 主入口 |
| `Toolbar` | 左侧工具栏 | 模块 `Jxq/` |
| `Workspace` | 中央工作区 | 模块 `15ck` |
| `Stage` | 右侧舞台 | 懒加载模块 |
| `PropertyPanel` | 属性面板 | 模块 `pp/G` |
| `CodePreview` | 代码预览 | 懒加载模块 |
| `Debugger` | 调试器 | 懒加载模块 |

### 2.4 数据流向

```
Redux Store → mapStateToProps → React 组件 → 渲染 UI
     ↑                                      ↓
     └──────── dispatch(Action) ←── 用户交互 ←┘
```

## 3. 状态管理层

### 3.1 职责

状态管理层负责：
- 管理全局状态
- 处理状态变更
- 管理副作用（异步操作）
- 提供状态订阅机制

### 3.2 技术栈

- **Redux 4.x**：状态管理
- **Redux-Saga**：副作用管理
- **Reselect**：状态选择器（可选）

### 3.3 核心概念

| 概念 | 说明 |
|------|------|
| **Store** | 全局状态树 |
| **Action** | 状态变更的描述对象 |
| **Reducer** | 纯函数，处理 Action 并返回新状态 |
| **Saga** | 处理异步操作的 Generator 函数 |
| **Dispatch** | 发送 Action 触发状态变更 |

### 3.4 Store 结构

```javascript
{
    project_state: { ... },      // 项目状态
    theatre_state: { ... },      // 剧院状态（角色、场景）
    variable_state: { ... },     // 变量状态
    audio_state: { ... },        // 音频状态
    midi_state: { ... },         // MIDI 状态
    toolbox_state: { ... },      // 工具栏状态
    snackbar_state: { ... },     // 提示消息状态
    save_state: { ... },         // 保存状态
    userinfo_state: { ... },     // 用户状态
    error_state: { ... },        // 错误状态
    hardware_state: { ... },     // 硬件状态
    ai_state: { ... },           // AI 状态
    // ... 更多状态
}
```

## 4. Blockly 引擎层

### 4.1 职责

Blockly 引擎层负责：
- 提供积木编程环境
- 管理工作区
- 处理积木拖拽和连接
- 序列化/反序列化积木数据

### 4.2 核心组件

| 组件 | 功能 |
|------|------|
| **Workspace** | 工作区，积木编程的主要区域 |
| **Toolbox** | 工具箱，左侧的积木分类面板 |
| **Flyout** | 积木选择面板，显示分类下的积木 |
| **Block** | 积木，编程的基本单元 |
| **Connection** | 连接点，积木之间的连接 |
| **Field** | 字段，积木上的输入框、下拉框等 |
| **Events** | 事件系统，追踪工作区变化 |

### 4.3 积木类型

| 类型 | 说明 | 连接口 |
|------|------|--------|
| **语句积木** | 执行操作 | 上下方形连接口 |
| **值积木** | 返回值 | 圆形连接口 |
| **帽子积木** | 事件触发器 | 只有下方连接口 |
| **变体积木** | 可动态增减输入 | 根据定义 |

### 4.4 关键 API

```javascript
// 工作区操作
Blockly.inject(element, config)           // 初始化工作区
Blockly.mainWorkspace                     // 获取主工作区
Blockly.json.workspace_to_json(workspace) // 序列化
Blockly.json.json_to_workspace(json, ws)  // 反序列化

// 积木操作
workspace.get_block_by_id(id)            // 获取积木
workspace.get_all_blocks()                // 获取所有积木
workspace.clean_up()                      // 整理积木

// 事件操作
workspace.add_change_listener(callback)   // 添加监听
Blockly.events.fire(event)                // 触发事件
Blockly.events.disable()                  // 禁用事件
```

## 5. 执行引擎层

### 5.1 职责

执行引擎层负责：
- 编译积木为字节码
- 执行字节码程序
- 管理 Domain Function
- 处理运行时错误

### 5.2 核心组件

| 组件 | 功能 |
|------|------|
| **Registry** | 注册中心，管理所有 Domain Function |
| **OptiCompiler** | 字节码编译器，将积木编译为字节码 |
| **OptiRunner** | 字节码解释器，执行字节码程序 |
| **DebugRunner** | 调试模式解释器 |
| **RuntimeManager** | 运行时管理器 |
| **BlockPool** | 积木对象池 |

### 5.3 执行流程

```
积木树 (Blockly Blocks)
  ↓
OptiCompiler 编译
  ↓
字节码程序 (OptiProgram)
  ↓
OptiRunner 执行
  ↓
调用 Domain Function
  ↓
返回结果 / 更新状态
```

### 5.4 关键概念

| 概念 | 说明 |
|------|------|
| **Domain Function** | 积木的运行时实现函数 |
| **Registry** | 单例注册中心，管理 Domain Function 索引 |
| **OptiProgram** | 编译后的字节码程序 |
| **OptiFrame** | 执行帧，保存执行上下文 |
| **Interpreter** | 解释器实例 |

## 6. 基础设施层

### 6.1 职责

基础设施层负责：
- 依赖注入
- 事件通信
- 错误处理
- 国际化
- 主题管理
- 日志记录

### 6.2 核心组件

| 组件 | 功能 | 说明 |
|------|------|------|
| **DI 容器** | 依赖注入 | 使用 Inversify |
| **EventBus** | 事件总线 | 组件间通信 |
| **Ohno** | 错误处理 | 错误定义和处理 |
| **i18n** | 国际化 | 多语言支持 |
| **Theme** | 主题管理 | 样式主题 |
| **Log** | 日志系统 | 日志记录 |

### 6.3 依赖注入

K4 使用 Inversify 实现依赖注入：

```javascript
// 绑定
container.bind(BINDING.Registry).to(RegistryImpl).inSingletonScope();
container.bind(BINDING.OptiCompiler).to(OptiCompilerImpl).inSingletonScope();

// 注入
@injectable()
class MyClass {
    constructor(
        @inject(BINDING.Registry) private registry: Registry,
        @inject(BINDING.OptiCompiler) private compiler: OptiCompiler
    ) {}
}
```

### 6.4 EventBus

```javascript
class EventBus {
    on(event, callback) { ... }    // 监听事件
    off(event, callback) { ... }   // 取消监听
    emit(event, data) { ... }      // 触发事件
    once(event, callback) { ... }  // 一次性监听
}
```

## 7. 层间依赖关系

### 7.1 依赖规则

```
UI 层 ──────→ 状态管理层
  │               │
  │               ↓
  └────────→ Blockly 引擎层
                  │
                  ↓
            执行引擎层
                  │
                  ↓
            基础设施层
```

**依赖规则**：
- 上层可以依赖下层
- 下层不能依赖上层
- 同层之间尽量避免依赖

### 7.2 数据流向

```
用户操作 → UI 层 → dispatch(Action) → 状态管理层
                                         ↓
                                    Reducer 处理
                                         ↓
                                    Store 更新
                                         ↓
                                    UI 层重新渲染
```

### 7.3 积木执行流

```
用户点击运行 → UI 层触发
                  ↓
            状态管理层 dispatch(runtime_start)
                  ↓
            RuntimeManager 启动
                  ↓
            Blockly 引擎层获取积木树
                  ↓
            执行引擎层编译并执行
                  ↓
            调用 Domain Function
                  ↓
            更新状态 → UI 层显示结果
```
