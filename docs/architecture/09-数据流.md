## 1. 数据流概述

### 1.1 核心数据流

```
┌─────────────────────────────────────────────────────────────┐
│                    K4 Ultra 数据流                           │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  用户操作                                                   │
│     │                                                       │
│     ├─→ 积木拖拽 ──→ Blockly 事件 ──→ Redux 状态更新        │
│     │                                                       │
│     ├─→ 点击运行 ──→ 编译积木 ──→ 执行字节码 ──→ 更新舞台   │
│     │                                                       │
│     ├─→ 点击保存 ──→ 收集数据 ──→ 序列化 ──→ 写入文件       │
│     │                                                       │
│     └─→ 协作编辑 ──→ OT 操作 ──→ Socket.IO ──→ 同步状态    │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 数据流类型

| 类型 | 说明 | 示例 |
|------|------|------|
| **用户交互流** | 用户操作触发的数据流 | 积木拖拽、点击按钮 |
| **执行流** | 积木执行的数据流 | 编译、执行、更新 |
| **持久化流** | 数据保存和加载 | 保存项目、加载项目 |
| **同步流** | 多端数据同步 | 协作编辑、云变量 |
| **事件流** | 系统事件传播 | 错误、通知、状态变更 |

## 2. 积木拖拽流程

### 2.1 完整流程

```
用户从 Flyout 拖拽积木
  │
  ├─→ 1. 检测拖拽开始
  │     ├─→ 触发 START_DRAG 事件
  │     └─→ 创建积木副本
  │
  ├─→ 2. 更新拖拽位置
  │     ├─→ 跟踪鼠标/触摸位置
  │     ├─→ 更新积木位置
  │     └─→ 检测连接点
  │
  ├─→ 3. 检测连接
  │     ├─→ 扫描附近的连接点
  │     ├─→ 检查连接类型匹配
  │     └─→ 显示连接预览
  │
  ├─→ 4. 放置积木
  │     ├─→ 触发 END_DRAG 事件
  │     ├─→ 建立连接
  │     └─→ 更新积木树
  │
  ├─→ 5. 触发 CREATE 事件
  │     ├─→ 记录操作历史（撤销/重做）
  │     └─→ 通知状态更新
  │
  └─→ 6. 更新 Redux Store
        ├─→ dispatch(workspace_change)
        └─→ 触发自动保存
```

### 2.2 关键事件

| 事件 | 触发时机 | 说明 |
|------|----------|------|
| `START_DRAG` | 开始拖拽 | 记录拖拽开始状态 |
| `DRAG` | 拖拽中 | 更新位置，检测连接 |
| `END_DRAG` | 结束拖拽 | 建立连接，更新树 |
| `CREATE` | 积木创建 | 记录历史，通知状态 |
| `MOVE` | 积木移动 | 更新位置 |
| `DELETE` | 积木删除 | 移除积木 |
| `CHANGE` | 积木变更 | 更新属性 |

### 2.3 连接检测

```javascript
// 连接类型
CONNECTION_TYPE = {
    NEXT_STATEMENT: 1,      // 下方连接（语句）
    PREVIOUS_STATEMENT: 2,  // 上方连接（语句）
    INPUT_VALUE: 3,         // 输入连接（值）
    OUTPUT_VALUE: 4         // 输出连接（值）
}

// 连接检测逻辑
function checkConnection(draggedBlock, targetConnection) {
    // 检查类型匹配
    if (!isTypeCompatible(draggedBlock.type, targetConnection.type)) {
        return false;
    }
    
    // 检查距离
    const distance = getDistance(draggedBlock, targetConnection);
    if (distance > SNAP_RADIUS) {
        return false;
    }
    
    // 检查是否会导致循环
    if (wouldCreateCycle(draggedBlock, targetConnection)) {
        return false;
    }
    
    return true;
}
```

## 3. 积木执行流程

### 3.1 完整流程

```
用户点击运行按钮
  │
  ├─→ 1. 触发运行
  │     ├─→ Redux dispatch(runtime_start)
  │     └─→ RuntimeManager 启动
  │
  ├─→ 2. 收集帽子积木
  │     ├─→ 遍历工作区
  │     ├─→ 找到所有帽子积木
  │     └─→ 为每个帽子积木创建 Interpreter
  │
  ├─→ 3. 编译积木树
  │     ├─→ OptiCompiler 遍历积木树
  │     ├─→ 对每个积木:
  │     │     ├─→ 查找 Domain Function 索引
  │     │     ├─→ 生成字节码指令
  │     │     └─→ 处理连接关系
  │     └─→ 输出字节码程序 (OptiProgram)
  │
  ├─→ 4. 执行字节码
  │     ├─→ OptiRunner 创建执行帧
  │     ├─→ 逐条执行指令:
  │     │     ├─→ call_domain_function:
  │     │     │     ├─→ 用索引获取 Domain Function
  │     │     │     ├─→ 准备参数
  │     │     │     ├─→ 调用函数
  │     │     │     └─→ 处理返回值
  │     │     ├─→ jump / jump_if:
  │     │     │     └─→ 控制流跳转
  │     │     └─→ yield:
  │     │           └─→ 协程让出
  │     └─→ 处理协程调度
  │
  ├─→ 5. 更新状态
  │     ├─→ 更新角色位置/外观
  │     ├─→ 更新变量值
  │     ├─→ 播放声音
  │     └─→ 触发动画
  │
  └─→ 6. 运行结束
        ├─→ Redux dispatch(runtime_stop)
        └─→ 清理资源
```

### 3.2 解释器调度

```
RuntimeManager 调度循环
  │
  ├─→ 按优先级排序解释器
  │
  ├─→ 对每个解释器:
  │     ├─→ 检查是否可运行
  │     ├─→ 执行一步
  │     ├─→ 处理 yield
  │     └─→ 更新状态
  │
  ├─→ 处理事件队列
  │     ├─→ 广播事件
  │     └─→ 消息传递
  │
  └─→ 请求下一帧
        └─→ requestAnimationFrame
```

### 3.3 协程处理

```javascript
// 协程状态
{
    is_yielding: false,      // 是否正在 yield
    yield_value: null,       // yield 值
    yield_reason: null,      // yield 原因（wait、animation、event）
    resume_frame: null       // 恢复帧
}

// yield 处理
function handleYield(interpreter, result) {
    // 保存执行状态
    interpreter.is_yielding = true;
    interpreter.yield_value = result.value;
    interpreter.yield_reason = result.reason;
    
    // 根据原因处理
    switch (result.reason) {
        case 'wait':
            // 等待指定时间后恢复
            setTimeout(() => resumeInterpreter(interpreter), result.value);
            break;
        
        case 'animation':
            // 动画完成后恢复
            registerAnimationCallback(interpreter, result.value);
            break;
        
        case 'event':
            // 事件触发后恢复
            registerEventCallback(interpreter, result.value);
            break;
    }
}
```

| 运行启动 | `runtime_start` |
| RuntimeManager | `RuntimeManagerImpl` |
| OptiCompiler | `OptiCompilerImpl` |
| OptiRunner | `OptiRunner` |
| Domain Function 调用 | `step_call_domain_function` |
| 协程处理 | `yield` |

## 4. 项目保存流程

### 4.1 完整流程

```
用户点击保存（或自动保存触发）
  │
  ├─→ 1. 触发保存请求
  │     ├─→ Redux dispatch(action_send_save_request)
  │     └─→ Saga 捕获 action
  │
  ├─→ 2. 收集项目数据
  │     ├─→ 收集工作区积木数据
  │     │     └─→ Blockly.json.workspace_to_json(workspace)
  │     │
  │     ├─→ 收集舞台状态
  │     │     ├─→ 角色数据
  │     │     ├─→ 场景数据
  │     │     └─→ 背景数据
  │     │
  │     ├─→ 收集变量和列表
  │     │     ├─→ 全局变量
  │     │     ├─→ 角色变量
  │     │     └─→ 列表数据
  │     │
  │     ├─→ 收集音频资源
  │     │     ├─→ 音频文件引用
  │     │     └─→ 音频配置
  │     │
  │     └─→ 收集其他数据
  │           ├─→ MIDI 数据
  │           ├─→ 云变量
  │           └─→ 扩展数据
  │
  ├─→ 3. 序列化为 JSON
  │     ├─→ 构建项目对象
  │     ├─→ JSON.stringify 序列化
  │     └─→ 压缩（可选）
  │
  ├─→ 4. 保存到目标
  │     ├─→ 本地文件：
  │     │     └─→ fs.writeFileSync(path, data)
  │     │
  │     ├─→ 服务器：
  │     │     └─→ axios.post(url, data)
  │     │
  │     └─→ 云存储：
  │           └─→ 云 API 调用
  │
  ├─→ 5. 更新保存状态
  │     ├─→ Redux dispatch(set_save_state)
  │     │     └─→ { is_saving: false, last_save_time: Date.now() }
  │     └─→ 显示保存成功提示
  │
  └─→ 6. 触发后续操作
        ├─→ 更新项目版本
        ├─→ 通知协作用户
        └─→ 记录历史版本
```

### 4.3 自动保存

```javascript
// 自动保存 Saga
function* autoSaveSaga() {
    while (true) {
        // 等待 30 秒
        yield delay(30000);
        
        // 检查是否需要保存
        const isModified = yield select(state => state.project_state.is_modified);
        
        if (isModified) {
            // 触发保存
            yield put({ type: 'action_send_save_request' });
        }
    }
}
```

## 6. 协作同步流程

### 6.1 完整流程

```
用户加入协作房间
  │
  ├─→ 1. 建立连接
  │     ├─→ Socket.IO 连接服务器
  │     ├─→ 发送 join_room 请求
  │     └─→ 接收房间信息
  │
  ├─→ 2. 同步初始状态
  │     ├─→ 接收当前项目数据
  │     ├─→ 加载项目到本地
  │     └─→ 显示协作用户列表
  │
  ├─→ 3. 本地编辑
  │     ├─→ 用户编辑积木
  │     ├─→ 生成 OT 操作
  │     └─→ 发送操作到服务器
  │
  ├─→ 4. 接收远程操作
  │     ├─→ 接收其他用户的操作
  │     ├─→ OT 转换操作
  │     └─→ 应用到本地
  │
  ├─→ 5. 冲突解决
  │     ├─→ 检测冲突
  │     ├─→ OT 转换
  │     └─→ 合并操作
  │
  └─→ 6. 离开房间
        ├─→ 发送 leave_room 请求

        └─→ 清理协作状态
```

## 7. 状态更新流程

### 7.1 Redux 状态更新

```
用户操作
  │
  ├─→ dispatch(Action)
  │
  ├─→ Reducer 处理
  │     ├─→ 接收当前状态和 Action
  │     ├─→ 根据 Action 类型处理
  │     └─→ 返回新状态
  │
  ├─→ Store 更新
  │     └─→ 通知所有订阅者
  │
  └─→ 组件重新渲染
        ├─→ mapStateToProps 执行
        └─→ render 执行
```

### 7.2 状态更新示例

```javascript
// 1. 用户点击角色
function handleActorClick(actorId) {
    // 2. dispatch Action
    store.dispatch({
        type: 'action_select_entity',
        payload: actorId
    });
}

// 3. Reducer 处理
function theatreReducer(state, action) {
    switch (action.type) {
        case 'action_select_entity':
            return {
                ...state,
                current_entity: action.payload
            };
        default:
            return state;
    }
}

// 4. 组件重新渲染
const mapStateToProps = (state) => ({
    currentEntity: state.theatre_state.current_entity
});

// 5. 渲染
function ActorList({ currentEntity, actors }) {
    return (
        <div>
            {Object.values(actors).map(actor => (
                <ActorItem
                    key={actor.id}
                    actor={actor}
                    isSelected={actor.id === currentEntity}
                />
            ))}
        </div>
    );
}
```

## 8. 错误处理流程

### 8.1 完整流程

```
错误发生
  │
  ├─→ 1. 捕获错误
  │     ├─→ try/catch 捕获
  │     ├─→ 错误边界捕获
  │     └─→ 全局错误处理
  │
  ├─→ 2. 创建错误对象
  │     ├─→ 错误类型
  │     ├─→ 错误消息
  │     ├─→ 错误堆栈
  │     └─→ 上下文信息
  │
  ├─→ 3. 记录错误日志
  │     ├─→ 控制台输出
  │     ├─→ 错误上报
  │     └─→ 本地存储
  │
  ├─→ 4. 显示错误 UI
  │     ├─→ 显示错误对话框
  │     ├─→ 高亮出错积木（如果是积木错误）
  │     └─→ 显示错误提示
  │
  └─→ 5. 恢复或停止
        ├─→ 如果可恢复：
        │     └─→ 恢复执行
        └─→ 如果不可恢复：
              └─→ 停止运行，清理资源
```

### 8.2 错误类型

| 类型 | 说明 | 处理方式 |
|------|------|----------|
| **系统错误** | 内部错误 | 显示通用错误消息 |
| **积木错误** | Domain Function 错误 | 高亮积木，显示详情 |
| **网络错误** | 请求失败 | 显示重试按钮 |
| **权限错误** | 无权限 | 显示权限提示 |
| **验证错误** | 数据验证失败 | 高亮错误字段 |

### 8.3 错误边界

```javascript
class ErrorBoundary extends React.Component {
    constructor(props) {
        super(props);
        this.state = { hasError: false, error: null };
    }
    
    static getDerivedStateFromError(error) {
        return { hasError: true, error };
    }
    
    componentDidCatch(error, errorInfo) {
        // 记录错误
        console.error('ErrorBoundary caught:', error, errorInfo);
        
        // 上报错误
        reportError(error, errorInfo);
    }
    
    render() {
        if (this.state.hasError) {
            return (
                <div className="error-boundary">
                    <h2>出现错误</h2>
                    <p>{this.state.error.message}</p>
                    <button onClick={() => this.setState({ hasError: false })}>
                        重试
                    </button>
                </div>
            );
        }
        
        return this.props.children;
    }
}
```
