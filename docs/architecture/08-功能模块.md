## 2. 工具栏系统

### 2.1 功能说明

工具栏是左侧的积木分类面板，用于组织和选择积木。

### 2.2 分类定义

工具栏分类在两处定义（`function u()` 和 `function c()` 内）：

```javascript
{
    events: {
        category_name: Blockly.Msg.events,    // 分类显示名
        color: Blockly.Msg.EVENTS_HUE,        // 分类颜色
        icon: a.a.event,                       // 分类图标
        id: s.c.events.id,                     // 分类 ID
        blocks: [                              // 包含的积木类型列表
            "start_on_click",
            "sprite_on_tap",
            "on_swipe",
            // ... 更多积木
        ]
    },
    // ... 更多分类
}
```

## 3. 积木定义系统

### 3.1 功能说明

积木定义系统负责定义积木的外观、参数和行为。

### 3.2 积木 JSON 定义

积木通过 JSON 数组批量定义，搜索 `define_blocks_with_json_array`：

```javascript
e.define_blocks_with_json_array([
    {
        type: "motion_movesteps",               // 积木类型名（唯一）
        message0: Blockly.Msg.MOTION_MOVESTEPS, // 积木显示文本
        args0: [                                 // 参数定义
            {
                type: "input_value",
                name: "STEPS",
                check: "Number"
            }
        ],
        previousStatement: null,                 // 上方连接
        nextStatement: null,                     // 下方连接
        colour: Blockly.Msg.MOTION_HUE,         // 颜色
        inputsInline: !0,                        // 内联输入
        tooltip: Blockly.Msg.MOTION_MOVESTEPS_TOOLTIP
    },
    // ... 更多积木定义
]);
```

### 3.4 参数类型

| 参数类型 | JSON 写法 | 说明 |
|---------|----------|------|
| 文本输入 | `{type:"input_value", name:"PATH", check:"String"}` | 连接一个值积木 |
| 数字输入 | `{type:"input_value", name:"NUM", check:"Number"}` | 连接一个数值积木 |
| 布尔输入 | `{type:"input_value", name:"BOOL", check:"Boolean"}` | 连接一个布尔积木 |
| 下拉选择 | `{type:"field_dropdown", name:"MODE", options:[...]}` | 固定选项列表 |
| 文本字段 | `{type:"field_input", name:"TEXT", text:"默认值"}` | 可编辑文本 |
| 数字字段 | `{type:"field_number", name:"NUM", value:0}` | 数字输入框 |
| 颜色字段 | `{type:"field_colour", name:"COLOR", colour:"#ff0000"}` | 颜色选择器 |
| 变量字段 | `{type:"field_variable", name:"VAR", variable:"myVar"}` | 变量选择器 |
| 图片字段 | `{type:"field_image", name:"IMG", src:"icon.png"}` | 图片显示 |

### 3.5 输出/语句类型

| 用途 | 写法 | 说明 |
|------|------|------|
| 值积木 | `output: "String"` / `"Number"` / `"Boolean"` / `"Array"` | 圆形连接口 |
| 语句积木 | `previousStatement: null, nextStatement: null` | 方形连接口 |
| 帽子积木 | `nextStatement: null`（无 previousStatement） | 只有下方连接口 |

### 3.6 JS Generator 注册

搜索 `blink.js_generator.register`：

```javascript
// 值积木 Generator
gen.register("motion_movesteps", function(block) {
    var steps = gen.value_to_code(block, "STEPS", gen.ORDER_NONE) || "10";
    return ["move(" + steps + ")", gen.ORDER_FUNCTION_CALL];
});

// 语句积木 Generator
gen.register("control_repeat", function(block) {
    var times = gen.value_to_code(block, "TIMES", gen.ORDER_NONE) || "10";
    var branch = gen.statement_to_code(block, "SUBSTACK");
    return "for (var i = 0; i < " + times + "; i++) {\n" + branch + "}\n";
});
```

### 3.7 Domain Function 注册

搜索 `get_domain_functions`：

```javascript
// 在 v 对象中注入
var v = {
    motion_movesteps: function(args) {
        var steps = args.STEPS || 0;
        // 移动角色
        return moveActor(steps);
    }
};
return v;
```

## 4. 扩展系统

### 4.1 传统魔改方式

直接修改 `kitten.822d814413fb10654fde.js`，需要修改 5 处：

1. 分类定义
2. 积木 JSON 定义
3. Domain Function 注册
4. JS Generator 注册
5. Heart IIFE 备份注册

**详细步骤参见**：`docs/K4魔改开发手册.md` 第 7 节

### 4.2 Blink 扩展系统（推荐）

Blink 扩展系统采用 `.k4ultra` 格式，零侵入，可打包分发。

**扩展包结构**：
```
my-extension.k4ultra          ← ZIP 包，改后缀为 .k4ultra
├── manifest.json             ← 扩展元数据（必需）
├── blocks.json               ← 积木外观定义
├── runtime.js                ← Domain Function（运行时执行逻辑）
├── generator.js              ← JS 代码生成器
└── resources/                ← 资源文件（图标等）
```

**manifest.json 格式**：
```json
{
    "name": "my-extension",
    "version": "1.0.0",
    "description": "扩展描述",
    "author": "作者名",
    "blink": {
        "version": "1",
        "min_engine": "4.0"
    },
    "category": {
        "name": "分类名称",
        "color": "#FF6B35",
        "icon": "resources/icon.png"
    },
    "entries": {
        "blocks": "blocks.json",
        "runtime": "runtime.js",
        "generator": "generator.js"
    },
    "security": {
        "level": "safe",
        "permissions": []
    }
}
```

### 4.3 Bridge API

```javascript
window.__k4 = {
    blocks: {
        define(jsonArray)           // 注册积木外观
    },
    generator: {
        register(type, fn)          // 注册单个代码生成器
        registerBatch(obj)          // 批量注册
    },
    runtime: {
        register(id, fn)            // 注册单个 domain function
        registerBatch(obj)          // 批量注册
    },
    security: {
        wrap(opId, desc, fn)        // 包装为需授权函数
    }
};
```

### 4.4 扩展加载流程

```
扩展加载器启动
  │
  ├─→ 扫描 extensions/ 目录
  │
  ├─→ 发现 .k4ultra 文件
  │
  ├─→ 解压 ZIP 文件
  │
  ├─→ 读取 manifest.json
  │
  ├─→ 注册积木 (blocks.json)
  │
  ├─→ 注册 Domain Function (runtime.js)
  │
  ├─→ 注册 Generator (generator.js)
  │
  └─→ 扩展加载完成
```

**详细步骤参见**：`docs/K4-Blink扩展开发指南.md`

## 7. 协作系统

### 7.1 OT 算法

K4 使用 OT（Operational Transformation）算法实现多人协作：

- **操作转换**：将远程操作转换为本地可执行的操作
- **冲突解决**：处理并发编辑冲突
- **状态同步**：保持所有客户端状态一致

### 7.2 Socket.IO 通信

```javascript
// 事件类型
- "join_room"：加入协作房间
- "leave_room"：离开协作房间
- "operation"：发送操作
- "ack"：确认操作
- "restore"：恢复历史版本
```

### 7.3 协作状态

```javascript
{
    is_coll_work: true,           // 是否协作模式
    coll_users: [ ... ],          // 协作用户列表
    socket: null,                 // Socket.IO 连接
    allow_send_or_receive: true   // 是否允许收发操作
}
```

## 12. 错误处理系统

### 12.1 错误类型

```javascript
{
    system: {
        unknown_system_error: { unique_number: 0 },
        missing_domain_function: { unique_number: 1 },
        // ...
    },
    client: {
        domain_function_error: { unique_number: 0 }
    }
}
```

### 12.2 错误处理流程

```
错误发生
  → 捕获错误
  → 创建错误对象
  → 显示错误对话框
  → 记录错误日志
  → 停止运行（如果是运行时错误）
```

### 12.3 错误对话框

- **系统错误**：显示"联系技术喵或班主任"
- **积木错误**：高亮出错的积木
- **网络错误**：显示重试按钮
