## 1. 项目概述

### 1.1 产品定位

K4 Ultra 是基于编程猫 Kitten4 编辑器的魔改版本，定位为专业游戏引擎。这是一个图形化编程编辑器，支持积木编程，允许用户通过拖拽积木块来创建程序。

### 1.2 核心特性

- **图形化积木编程环境**：基于 Google Blockly 引擎
- **双层执行架构**：JS Generator（代码预览）+ OptiCompiler/OptiRunner（字节码执行）
- **扩展系统**：传统魔改 + Blink 扩展系统（.k4ultra 格式）
- **内置系统 API 积木**：文件操作、剪贴板、系统信息等
- **离线模式支持**：Electron 桌面应用
- **实时协作**：基于 OT 算法的多人协作
- **AI 集成**：AI 实验室、AI 游戏模块
- **硬件扩展**：Arduino、micro:bit、Weeemake 等

### 1.3 文件结构

### 3.3 双层执行架构（关键）

K4 的积木执行系统采用**双层架构**：

```
┌─────────────────────────────────────────────────────────────┐
│                    K4 积木执行双层架构                        │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  ┌─────────────┐     ┌──────────────┐     ┌──────────────┐ │
│  │ 积木定义     │ ──→ │ JS Generator  │ ──→ │ 代码预览      │ │
│  │ (JSON)      │     │ (代码生成器)   │     │ ("看代码")    │ │
│  └─────────────┘     └──────────────┘     └──────────────┘ │
│         │                                               ▲   │
│         │              ★ 主执行路径 ★                    │   │
│         ↓                                               │   │
│  ┌─────────────┐     ┌──────────────┐     ┌──────────────┐ │
│  │ 积木定义     │ ──→ │ OptiCompiler  │ ──→ │ OptiRunner   │ │
│  │ (JSON)      │     │ (字节码编译器)  │     │ (字节码解释器)│ │
│  └─────────────┘     └──────────────┘     └──────────────┘ │
│                              │                    │        │
│                              ▼                    ▼        │
│                    ┌──────────────────────────────┐        │
│                    │     Registry (单例)           │        │
│                    │  domain_function_index[]      │        │
│                    │  domain_function_list[]       │        │
│                    └──────────────────────────────┘        │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**关键概念**：

| 概念 | 说明 |
|------|------|
| **Domain Function** | 积木的运行时实现函数，接收参数并返回结果 |
| **Registry** | 单例注册中心，管理所有 domain function 的索引映射 |
| **OptiCompiler** | 将积木编译为字节码，通过 block.type 查找 domain function 索引 |
| **OptiRunner** | 执行字节码时，用数字索引调用对应的 domain function |
| **JS Generator** | 生成可读 JS 代码字符串，仅用于"看代码"展示，不参与实际执行 |

## 6. Blockly 引擎层

### 6.1 核心概念

Blockly 是 Google 开源的图形化编程库，K4 在此基础上进行了大量扩展。

### 6.2 Workspace（工作区）

工作区是积木编程的主要区域，包含：

- **SVG 画布**：渲染积木的矢量图形
- **滚动系统**：支持平移和缩放
- **网格系统**：对齐辅助
- **撤销/重做**：操作历史管理

**关键代码位置**：
- 工作区初始化：`Blockly.inject(element, config)`
- 获取主工作区：`Blockly.mainWorkspace`
- 工作区序列化：`Blockly.json.workspace_to_json(workspace)`
- 工作区反序列化：`Blockly.json.json_to_workspace(json, workspace)`

### 6.3 Toolbox（工具箱）

工具箱是左侧的积木分类面板，包含：

- **分类列表**：事件、控制、运动等
- **Flyout 面板**：点击分类后显示的积木列表
- **搜索功能**：积木搜索

**关键代码位置**：
- 工具栏配置：搜索 `category_name: Blockly.Msg`
- 分类定义：搜索 `function u()` 和 `function c()`
- Flyout 组件：模块 `+UMZ`

### 6.4 Blocks（积木）

积木是编程的基本单元，分为：

- **语句积木**：有上下连接口，执行操作
- **值积木**：有圆形连接口，返回值
- **帽子积木**：只有下方连接口，事件触发器
- **变体积木**：可动态增减输入

**积木定义格式**：
```javascript
{
    type: "block_type_name",           // 积木类型名（唯一）
    message0: "移动 %1 步",            // 积木显示文本
    args0: [                           // 参数定义
        { type: "input_value", name: "STEPS", check: "Number" }
    ],
    previousStatement: null,           // 上方连接（Statement）
    nextStatement: null,               // 下方连接（Statement）
    colour: "#4C97FF",                 // 颜色
    tooltip: "让角色移动指定步数",     // 提示
    helpUrl: ""                        // 帮助链接
}
```

### 6.5 Events（事件系统）

Blockly 事件系统用于追踪工作区变化：

- `BlockEventType.UI`：UI 事件
- `BlockEventType.MOVE`：积木移动
- `BlockEventType.CHANGE`：积木属性变更
- `BlockEventType.DELETE`：积木删除
- `BlockEventType.CREATE`：积木创建
- `BlockEventType.START_DRAG`：开始拖拽
- `BlockEventType.END_DRAG`：结束拖拽

**关键代码位置**：
- 事件监听：`Blockly.mainWorkspace.add_change_listener(callback)`
- 事件触发：`Blockly.events.fire(event)`
- 事件禁用：`Blockly.events.disable()` / `Blockly.events.enable()`

## 7. 执行引擎层

### 7.1 Registry（注册中心）

Registry 是执行引擎的核心，管理所有 domain function 的注册和查找。

**数据结构**：
```javascript
class RegistryImpl {
    domain_function_index = {}    // { "积木类型": 索引 }
    domain_function_list = []     // [ 函数引用, ... ]
    domain_function_types = []    // [ "积木类型", ... ]
    
    register({ namespace, id, domain_function }) {
        // 注册 domain function
    }
    
    get_domain_function_index(type) {
        // 获取 domain function 索引
    }
    
    get_domain_function_list() {
        // 获取所有 domain function 列表
    }
}
```

**关键代码位置**：
- Registry 实现：搜索 `RegistryImpl`
- 注册方法：搜索 `registry.register`
- 单例绑定：搜索 `BINDING.Registry`

### 7.2 OptiCompiler（字节码编译器）

OptiCompiler 将积木树编译为字节码程序。

**编译流程**：
```
积木树 (Blockly Blocks)
  → 遍历积木
    → 对每个积木:
      → 查找 domain function 索引
      → 生成字节码指令
  → 输出字节码程序 (OptiProgram)
```

**关键代码位置**：
- 编译器实现：搜索 `OptiCompilerImpl`
- Domain Function 编译：搜索 `compile_domain_functions`
- 字节码关键字：搜索 `call_domain_function`

**字节码指令示例**：
```javascript
// 编译 "移动 10 步" 积木
[
    "X.call_domain_function",  // 指令类型
    "('motion_movesteps',",    // domain function 名称
    "0,",                       // domain function 索引
    "'block_123',",            // 积木 ID
    "null,",                   // 动态数据
    "entity_456);"             // 目标实体
]
```

### 7.3 OptiRunner（字节码解释器）

OptiRunner 执行 OptiCompiler 编译产生的字节码。

**执行流程**：
```
字节码程序
  → 创建执行帧 (OptiFrame)
  → 逐条执行指令
    → 遇到 call_domain_function 指令:
      → 用索引从 domain_function_list 获取函数
      → 调用函数，传入参数
      → 处理返回值
  → 处理协程 (yield)
  → 处理错误
```

**关键代码位置**：
- Runner 实现：搜索 `OptiRunner`
- Domain Function 调用：搜索 `step_call_domain_function`
- 协程处理：搜索 `yield`

### 7.4 Domain Function（运行时函数）

Domain Function 是积木的实际执行逻辑。

**标准签名**：
```javascript
function domain_function_name(args, interpreter_id, entity_id, util) {
    // args = 积木参数对象
    // interpreter_id = 解释器 ID
    // entity_id = 目标实体 ID
    // util = 工具函数对象
    
    try {
        // 业务逻辑
        return result;  // 值积木必须返回结果
    } catch(err) {
        // 错误处理，不要抛异常
        return default_value;
    }
}
```

**注册方式**：
```javascript
// 方式一：在 get_domain_functions 的 v 对象中注入
var v = {
    my_function: function(args) {
        return doSomething(args.PARAM);
    }
};
return v;

// 方式二：通过 Registry 注册
registry.register({
    namespace: "",
    id: "my_function",
    domain_function: function(args) {
        return doSomething(args.PARAM);
    }
});
```

### 7.5 DebugRunner（调试模式解释器）

DebugRunner 是 OptiRunner 的调试版本，支持：

- 断点调试
- 单步执行
- 变量监视
- 调用栈查看

**关键代码位置**：
- DebugRunner 实现：搜索 `DebugRunner`
- 调试模式切换：搜索 `debug_mode`

## 23. 数据流详解

### 23.1 积木拖拽流程

```
用户从 Flyout 拖拽积木
  → Blockly 检测拖拽开始 (START_DRAG)
  → 创建积木副本
  → 更新拖拽位置
  → 检测连接点
  → 放置积木 (END_DRAG)
  → 触发 CREATE 事件
  → 更新 Redux Store
  → 重新渲染
```

### 23.2 积木执行流程

```
用户点击运行
  → Redux dispatch(runtime_start)
  → RuntimeManager 启动
  → 收集所有帽子积木
  → 为每个帽子积木创建 Interpreter
  → OptiCompiler 编译积木树
  → OptiRunner 执行字节码
    → 调用 Domain Function
    → 处理协程 (yield)
    → 更新舞台状态
  → 运行结束
  → Redux dispatch(runtime_stop)
```

### 23.3 项目保存流程

```
用户点击保存
  → Redux dispatch(action_send_save_request)
  → Saga 捕获 action
  → 收集工作区数据 (Blockly.json.workspace_to_json)
  → 收集舞台状态
  → 收集变量和列表
  → 序列化为 JSON
  → 发送到服务器（或保存到本地）
  → 更新保存状态
  → 显示保存成功提示
```

### 23.4 协作同步流程

```
用户编辑积木
  → 触发 Blockly 事件
  → 转换为 OT 操作
  → 发送到服务器
  → 服务器广播给其他用户
  → 其他用户接收操作
  → 转换为本地操作
  → 应用到工作区
  → 重新渲染
```
